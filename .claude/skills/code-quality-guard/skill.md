# Code Quality Guard Skill

코드 품질을 자동으로 점검하고 문제를 수정하는 스킬입니다.

## 언제 이 스킬을 사용하나요?

- 코드 작성/수정 후 품질 검사가 필요할 때
- PR 생성 전 최종 품질 점검

## 검사 항목

1. **컴파일** - `./gradlew compileKotlin`
2. **코드 스타일** - `./gradlew ktlintCheck`
3. **테스트** - `./gradlew test`
4. **빌드** - `./gradlew build`

## 자동 수정 가능 항목

- **코드 스타일**: `./gradlew ktlintFormat`으로 자동 수정
- **import 정리**: 사용하지 않는 import 제거
- **공백/들여쓰기**: 자동 교정

## 수정 불가 항목 (수동 리뷰 필요)

- 비즈니스 로직 에러
- 아키텍처 위반
- N+1 쿼리 문제
- 보안 이슈

## 코드 리뷰 체크리스트

- [ ] TDD 원칙을 따라 테스트 코드가 먼저 작성되었는가?
- [ ] 모든 테스트가 통과하는가?
- [ ] 공통 유틸리티를 재사용했는가?
- [ ] 에러 처리가 적절하게 되어 있는가?
- [ ] 코드가 Kotlin 컨벤션을 따르는가?
- [ ] 불필요한 의존성이 추가되지 않았는가?
